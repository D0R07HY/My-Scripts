# Beginner Cyber Security Glossary

ไฟล์นี้คือพจนานุกรมคำศัพท์สำหรับมือใหม่

จุดประสงค์ของไฟล์นี้ไม่ใช่การให้คำจำกัดความแบบวิชาการเป๊ะที่สุด
แต่คือการช่วยให้คุณ "อ่านเอกสารในรีโปนี้แล้วเข้าใจมากขึ้น"

ถ้าเจอคำไหนใน roadmap, report, หรือ issue แล้วงง ให้กลับมาเปิดไฟล์นี้ได้เสมอ

## A

### `Artifact`

คือร่องรอยที่โปรแกรมทิ้งไว้หลังจากทำงาน

ตัวอย่าง:

- ไฟล์ที่ถูกสร้าง
- process ที่ถูกรัน
- log
- registry key
- network connection

คิดง่าย ๆ:

artifact คือ "หลักฐานที่เหลืออยู่"

### `Analysis`

คือการวิเคราะห์

ในโปรเจคนี้มักหมายถึงการดูว่าไฟล์หรือโปรแกรมหนึ่งตัวมีลักษณะหรือพฤติกรรมอย่างไร

## B

### `Behavior`

คือพฤติกรรมของโปรแกรมตอนทำงาน

ตัวอย่าง:

- สร้างไฟล์
- ลบไฟล์
- เปิด process ใหม่
- ติดต่อ network

### `Binary`

คือไฟล์โปรแกรมที่พร้อมรัน หรือไฟล์ที่เป็นข้อมูลระดับเครื่อง

เช่น `.exe`, `.dll`, `.class`

พูดง่าย ๆ:

เป็นไฟล์ที่ไม่ใช่ข้อความธรรมดาแบบ `.txt`

## C

### `Command Line`

คือคำสั่งที่ใช้เรียกโปรแกรมผ่าน terminal หรือ shell

ตัวอย่าง:

```powershell
python analysis\scripts\hash_report.py LICENSE
```

### `Compiled Artifact`

คือไฟล์ที่ได้จากการ compile โค้ด เช่น `.class` จาก Java

ในรีโป public ทั่วไป เรามักเก็บ source code มากกว่าเก็บ compiled artifact

### `Confidence`

คือระดับความมั่นใจของเราในสิ่งที่วิเคราะห์

เช่น:

- มั่นใจต่ำ
- มั่นใจปานกลาง
- มั่นใจสูง

ใช้บอกว่าข้อมูลที่เราพบน่าเชื่อถือแค่ไหน

## D

### `Defensive`

แปลว่าเชิงป้องกัน

ในบริบทนี้หมายถึงเรียนรู้เพื่อ:

- ตรวจจับ
- วิเคราะห์
- ป้องกัน

ไม่ใช่เพื่อโจมตี

### `Detection`

คือการตรวจจับสิ่งผิดปกติหรือสิ่งที่น่าสงสัย

เช่น:

- ถ้าเจอ string แบบนี้ ให้แจ้งเตือน
- ถ้าเจอ process แบบนี้ ให้บันทึก

### `Detection Engineering`

คือการออกแบบวิธีตรวจจับอย่างมีเหตุผล

ไม่ใช่แค่เดาสุ่ม แต่คิดว่า:

- จะจับจากอะไร
- จับด้วยข้อมูลจากไหน
- จะพลาดอะไรไหม
- จะมี false positive ไหม

### `Dynamic Analysis`

คือการวิเคราะห์ตอนที่ไฟล์หรือโปรแกรมกำลังทำงาน

เช่น:

- เปิดโปรแกรมแล้วดูว่ามันสร้างไฟล์อะไร
- ดูว่าเปิด process อะไร
- ดูว่ามี network connection ไหม

## F

### `False Positive`

คือการแจ้งเตือนผิด

ตัวอย่าง:

ระบบบอกว่าไฟล์นี้น่าสงสัย แต่จริง ๆ เป็นไฟล์ปกติ

### `File Hash`

คือค่าประจำตัวของไฟล์

เหมือนลายนิ้วมือของไฟล์

ตัวอย่าง:

- MD5
- SHA1
- SHA256

### `Forensics`

คือการสืบหาหลักฐานจากระบบหรืออุปกรณ์

ในระดับเริ่มต้น คุณอาจยังไม่ต้องลงลึกมาก แต่แนวคิดเรื่องการเก็บ artifact และบันทึก evidence มีความเกี่ยวข้องกัน

## H

### `Hash`

คือค่าที่คำนวณจากข้อมูลในไฟล์

ถ้าไฟล์เปลี่ยนแม้แต่นิดเดียว hash ก็เปลี่ยน

ใช้ทำอะไร:

- ระบุไฟล์
- เปรียบเทียบไฟล์
- บันทึกลง IOC

### `Host`

คือเครื่องหลักของคุณ

ถ้าใช้ VM:

- host = เครื่องจริง
- guest = เครื่องเสมือน

### `Host-only Network`

คือการตั้งค่า network ให้ VM คุยได้ในวงจำกัด

ปลอดภัยกว่าเอา VM ไปต่ออินเทอร์เน็ตตรง ๆ

## I

### `IOC`

ย่อมาจาก `Indicator of Compromise`

แปลว่า "ตัวบ่งชี้ว่าระบบอาจถูกโจมตีหรือมีสิ่งผิดปกติ"

ตัวอย่าง:

- hash
- domain
- IP
- file path
- process name

คิดง่าย ๆ:

IOC คือ "รอยเท้าที่น่าจะเกี่ยวกับเหตุผิดปกติ"

### `Isolation`

คือการแยก

ในโปรเจคนี้หมายถึงการแยก environment ทดลองออกจากเครื่องจริงหรือระบบจริง

## L

### `Lab`

คือสภาพแวดล้อมทดลอง

ในบริบทนี้คือ VM หรือ environment ที่แยกไว้ใช้สำหรับการเรียนรู้และทดสอบอย่างปลอดภัย

### `Log`

คือบันทึกเหตุการณ์ของระบบหรือโปรแกรม

เช่น:

- โปรแกรมเริ่มรันเมื่อไร
- process ไหนถูกสร้าง
- มี error อะไรเกิดขึ้น

## M

### `Malware`

คือโปรแกรมอันตราย

เช่น:

- virus
- worm
- ransomware
- spyware

### `Malware Analysis`

คือการศึกษาว่า malware ตัวหนึ่งทำอะไรและทิ้งร่องรอยอะไรไว้

### `Metadata`

คือข้อมูลประกอบของไฟล์

ตัวอย่าง:

- ชื่อไฟล์
- ขนาดไฟล์
- ชนิดไฟล์
- เวลาที่เกี่ยวข้อง

## N

### `Network Artifact`

คือร่องรอยที่เกี่ยวข้องกับ network

เช่น:

- IP
- domain
- URL
- port

## P

### `Payload`

คือส่วนของโปรแกรมที่ทำงานหลักบางอย่าง

ในเนื้อหาปลอดภัยของเรา คุณไม่จำเป็นต้องลงลึกกับ payload เชิงอันตราย

### `Persistence`

คือเทคนิคที่ทำให้โปรแกรมกลับมาทำงานอีกแม้รีสตาร์ตเครื่อง

คำนี้สำคัญเพราะมันมักเกี่ยวกับพฤติกรรมที่ไม่ปลอดภัย และไม่ใช่สิ่งที่เราจะสร้างในโปรเจคนี้

### `Portfolio`

คือผลงานที่รวบรวมไว้ให้คนอื่นดู

เช่น GitHub ของคุณ

### `Process`

คือโปรแกรมที่กำลังรันอยู่

เช่น เปิด Chrome ขึ้นมา ก็จะเกิด process ของ Chrome

### `Process Tree`

คือความสัมพันธ์ว่า process ไหนเปิด process ไหน

เช่น:

- `explorer.exe` เปิด `cmd.exe`
- `cmd.exe` เปิด `python.exe`

## R

### `Registry`

คือฐานข้อมูลการตั้งค่าของ Windows

หลายโปรแกรมจะเขียนค่าลง registry

### `Report`

คือเอกสารสรุปสิ่งที่คุณพบจากการวิเคราะห์

### `Rule`

คือกฎตรวจจับ

ตัวอย่าง:

- YARA rule
- Sigma rule

## S

### `Sample`

คือไฟล์ตัวอย่างที่นำมาศึกษา

สำหรับมือใหม่ sample ไม่จำเป็นต้องเป็น malware จริงเสมอไป

### `Sigma`

คือรูปแบบกฎตรวจจับที่ใช้กับ log

มักใช้กับ:

- process creation log
- command line
- event logs

### `Simulator`

คือโปรแกรมจำลองพฤติกรรมแบบปลอดภัย

ใช้สร้างสถานการณ์เพื่อฝึก analysis และ detection

### `Snapshot`

คือการบันทึกสถานะของ VM ณ เวลาหนึ่ง

ถ้าทดลองแล้วพัง ก็ย้อนกลับมาได้

### `Static Analysis`

คือการวิเคราะห์โดยยังไม่รันไฟล์

เช่น:

- ดู hash
- ดู strings
- ดู metadata

### `Strings`

คือข้อความที่ดึงออกมาจากไฟล์ได้

เช่น:

- URL
- path
- ชื่อคำสั่ง
- error message

## T

### `Triage`

คือการคัดกรองหรือดูเบื้องต้นว่าอะไรสำคัญ

ในงาน analysis หมายถึงการดูอย่างเร็ว ๆ ก่อนว่าจะเริ่มจับตรงไหน

## V

### `VM`

ย่อมาจาก `Virtual Machine`

คือคอมพิวเตอร์จำลอง

คุณสามารถมี Windows VM หรือ Linux VM อยู่ในเครื่องจริงได้

### `Volume of Data`

คือปริมาณข้อมูลที่ต้องดู

ในอนาคตเมื่อคุณทำ analysis มากขึ้น จะเริ่มเจอว่าต้องคัดกรองข้อมูลเยอะขึ้น

## W

### `Writeup`

คือบทความหรือรายงานอธิบายเคสหนึ่ง ๆ

เช่นบทวิเคราะห์ malware จากบริษัท security

### `Workflow`

คือขั้นตอนการทำงานที่ทำซ้ำได้

ตัวอย่าง workflow ง่าย ๆ:

1. ดู hash
2. ดึง strings
3. จดลง report
4. คิด detection

## Y

### `YARA`

คือรูปแบบกฎตรวจจับที่มักใช้กับไฟล์หรือ content

มักใช้กับ:

- strings
- byte patterns
- file content

พูดง่าย ๆ:

YARA คือกฎที่ช่วยบอกว่า "ไฟล์แบบนี้มีลักษณะน่าสงสัยไหม"

## สรุปคำที่ควรรู้ก่อนที่สุด

ถ้าคุณจะจำแค่ชุดแรกก่อน ให้เริ่มจากคำเหล่านี้:

- VM
- Snapshot
- Static Analysis
- Dynamic Analysis
- Hash
- Strings
- IOC
- Artifact
- YARA
- Sigma
- Simulator

ถ้าคำพวกนี้เริ่มชัด การอ่าน roadmap และการทำโปรเจคจะง่ายขึ้นมาก
